# Winning Ad Scripts — Close Read
_Source: ClickUp Video Ads list, briefs B0131 / B0142 / B0207 / B0216 / B0224 / B0230. B0108 and B0112 are absent from current ClickUp (older briefs)._

The point of this doc is to look at **what each line of each winning script is actually doing**, then surface the rhetorical patterns that repeat across all of them. No format / editor / strategist meta — just the words.

---

## B0142 — Founder anti-knockoff (winner: H3, ROAS 2.21)

### The four hook variants

| # | Hook | Winner? |
|---|---|---|
| H1 | _"My name is James. I'm the founder of Miner Forge Pro. We're killing our bestselling Bitcoin miner and it's going to make a lot of people very angry."_ | no |
| H2 | _"My name is James. I'm the founder of Miner Forge Pro. I'm about to destroy our bestselling Bitcoin miner."_ | no |
| H3 | _"I'm the founder of Miner Forge Pro and I'm about to make an announcement that changes everything."_ | **YES** ($5,848 @ 2.21) |
| H4 | _"My name is James. I'm the founder of Miner Forge Pro. And what I'm about to tell you is going to piss off a lot of people."_ | no |

**The single tell.** H3 is the only hook that **drops the name "James"** and **drops the word "kill / destroy / piss off"**. H1/H2/H4 all front-load an aggressive verb. H3 fronts the **role + an announcement frame** — softer, more news-anchor, more curiosity-gap. The losing hooks read like _"someone's about to vent."_ The winning hook reads like _"someone's about to break news."_

This is worth saying directly to the strategist: **for a founder-voice hook on this product, the news-anchor framing beats the angry-founder framing.** The viewer doesn't want to watch a tantrum; they want to find out what the announcement is.

### Body — sequence of moves

| Line | What the line is doing |
|---|---|
| _"We're killing the original Miner Forge Pro. Done. Never making another one."_ | Lands the announcement; pre-commits the founder to a costly action so the viewer trusts the next claim. |
| _"Why would we destroy our bestselling device?"_ | Self-asks the obvious question — gives the viewer permission to be skeptical. |
| _"Because knockoff artists have flooded the market…Nobody copies garbage."_ | **Reframes the threat into a compliment.** Knockoffs become evidence of product quality. This is the script's first proof beat — and it's entirely emotional, no on-chain data yet. |
| _"We took every penny of profit from the last two years…makes the original look like a toy."_ | Costly-signal #2: founder is sacrificing margin. Aligns the founder financially with the product, not the sale. |
| _"We invested $800,000 and 18 months working with actual Bitcoin engineers."_ | Specific numbers replace adjectives. $800K and 18 months are unfalsifiable but feel earned. |
| _"175% more powerful than anything in this price range."_ | The only product-spec claim — and it's wrapped in price-range qualifier so it can't be picked apart. |
| _"three simple steps: plug it in, connect Wi-Fi, start competing."_ | Setup is **always exactly three steps** in MinerForge scripts. Setup-friction objection killed in one line. |
| _"Those fake miners flooding Facebook? They show animated screens that look impressive. But check the blockchain? Nothing."_ | The pivot from "we're real" to **"here's how to falsify the others."** Hands the viewer the test. |
| _"Every single attempt is recorded on the Bitcoin blockchain. 144 real attempts every day at the full 3.125 Bitcoin block worth $330,000+."_ | First proof number drop. Specifics over claims. **3.125 BTC** (the post-halving block reward, accurate as of 2024-onward) is the kind of detail the audience checks for. |
| _"$59.99 instead of the regular $100…code 2026 at checkout for another 20% off. That's 62% off the normal price."_ | Anchored discount. The 62% is the headline; the $59.99 is the anchor. |
| _"This pricing is only for the first 1,000 people."_ | Scarcity, but bounded — "first 1,000" is concrete enough to feel real, vague enough to be unverifiable. |
| _"if it doesn't work exactly like I promised? Send it back within 90 days for every penny back. I'm taking all the risk."_ | Standard risk-reversal, voiced first-person ("I'm taking all the risk") — keeps it founder-coded. |
| _"Click below. It takes you straight to our verified site, not some sketchy reseller with counterfeits."_ | **CTA does double duty as anti-fake reassurance.** The click itself is framed as the safe choice. |
| _"Do it now before we sell out."_ | Closes the scarcity loop. |

### The script's spine

`Costly action → reframe the threat → invest $800K → 3-step setup → falsify the competitors → on-chain proof number → anchored discount → bounded scarcity → first-person guarantee → safe-click CTA → urgency.`

The body never breaks pose. The founder never asks the viewer to feel anything; he keeps producing **falsifiable signals.** The viewer is allowed to remain a skeptic the whole way through.

## B0224 — Self-skeptic on-chain proof (winner: H3, ROAS 2.05)

### Hook variants

| # | Hook | Winner? |
|---|---|---|
| H1 | _"I thought this was fake too. Then I pulled up the blockchain myself."_ + _"This is MinerForge Pro. It competes for full Bitcoin blocks worth around $300,000. Solo."_ | not winner |
| H2 | _"I was certain this was a scam. So I opened Block Explorer and started digging."_ + same product line | not winner |
| H3 | _"Everyone told me it was too good to be true. So I went straight to the blockchain to find out."_ + same product line | **YES** ($564 @ 2.05) |

**The single tell.** H1/H2 are first-person doubts the speaker held internally. H3 outsources the doubt to **other people** — _"Everyone told me…"_ The viewer is no longer being asked to identify with one skeptic; they're being shown a **social consensus that the speaker just disproved.** That's a much stronger setup because the viewer's own dismissive thought ("this sounds too good to be true") is exactly the "everyone" being quoted.

### Body — what each line does

| Line | What it's doing |
|---|---|
| _"Independent. No pool. No splitting."_ | Three-word product positioning. Sets the differentiator before the proof reel. |
| _"When I first heard about it I was certain it was a scam. So I opened Block Explorer and started digging through real on chain data."_ | Restates the hook's claim — re-anchors the skepticism POV in case the viewer landed mid-scroll. |
| _**"Block 91,612. Solo mined four hours ago. 3.125 Bitcoin. One miner. One wallet. Full reward."**_ | **The single most distinctive line in the entire MinerForge ad library.** Specific block number + specific timestamp + specific reward + denial of any shared payout. Whether or not the viewer verifies, the **claim shape** ("I'm telling you exact block numbers; go check") collapses the scam objection. |
| _"Here is another one. Block 92,618. Three hours ago. Solo mined again."_ | Second instance — proves the first wasn't cherry-picked. **Pattern of two, not one.** |
| _"These are not coming from industrial warehouses. These are coming from small devices exactly like this one. Sitting on a desk somewhere. Running quietly on 11 cents a month."_ | Closes the "but the warehouses must be doing this" objection in one line. Names the alternative (your desk) and prices it (11 cents/month). |
| _"The blockchain does not lie."_ | Proverb. The viewer's own pre-existing belief about Bitcoin gets weaponized in the script's favor. |
| _"144 shots at that block reward every single day. Automatically. Around the clock. From the moment you plug it in."_ | Frequency beat. "From the moment you plug it in" eliminates the wait-for-results objection. |
| _"Setup takes 60 seconds. Costs $1 a year to run."_ | Setup + cost in one sentence. **$1 a year** is the most aggressive version of the cost frame across all the scripts. |
| _"I stopped doubting and started running one."_ | Identification beat — the skeptic concedes. The viewer is invited to do the same. |
| _"The wins are real. The data is public. Anyone can verify it."_ | Three-clause permission to verify, mirroring the three-step setup elsewhere. |
| _"Up to 58% off right now with free shipping."_ | Discount drop. |
| _"Tap the link below. Proof is on the blockchain. Your call."_ | **CTA does not push.** "Your call." is the only CTA in the library that explicitly hands the decision back. For an audience trained to distrust pushy crypto offers, this is the right move. |

### The script's spine

`Outsourced doubt → product specificity (independent / no pool) → SPECIFIC BLOCK NUMBERS as falsifiable proof → second specific block to defeat cherry-picking → close the warehouse objection → proverb (blockchain doesn't lie) → frequency → cost → speaker converts → permission to verify → discount → "your call" CTA.`

The architectural choice that makes this script work: **B0224 is the only script in the set that names verifiable artifacts in the world** (block numbers). Every other script makes claims that the viewer would have to trust. B0224 makes claims the viewer can falsify if they want to. The buyer almost certainly won't check. The fact that they **could** is what converts them.

## B0230 — AI-chip first-person POV (winner: H2, ROAS 2.02)

### Hook variants (3 in brief)

| # | Hook | Winner? |
|---|---|---|
| H1 | _"Did you know you can compete for $300,000 in Bitcoin every single day from your desk?"_ | $93 spend, ROAS 4.60 (tiny sample) |
| H2 | _"Most people plug in MinerForge Pro and let it run. But very few know what is actually happening inside it. Let me show you."_ | **YES** ($752 @ 2.02) |
| H3 | _"This is what $300,000 worth of Bitcoin mining looks like from a $60 device. Here is what is working inside it around the clock."_ | $452 / $0 — broken |

**The single tell.** H1 pitches the product directly ("you can compete for $300K"). H3 frames the product as a curiosity ("here is what's inside"). Both got ignored or didn't convert. H2 frames the script as a **secret being shared with the viewer** ("very few know what is actually happening inside it. Let me show you.") The hook isn't selling the product — it's offering insider access. That's the move that converts here.

### Body — what each line does

The body is one of the most disciplined pieces of copy in the set. Seven beats, each opening with **"I am ___"**, each beat one product attribute personified by the AI chip.

| Beat | Line | What it's claiming |
|---|---|---|
| 1 | _"I am an AI chip. I power every solo mining attempt MinerForge Pro fires at the Bitcoin network. 144 times a day. Automatically. Without you lifting a finger."_ | Frequency. |
| 2 | _"I am solo mining. I hunt for the full block reward completely alone. No pool. No splitting. No sharing with thousands of strangers. If we win, every dollar goes straight to your wallet."_ | Differentiator (vs. pools). |
| 3 | _"I am blockchain verification. Every attempt I submit is real. Recorded. Public. Anyone can pull up the ledger and confirm I am working. No simulation. No fake activity. All on chain."_ | Falsifiability. |
| 4 | _"I am one watt of power. I cost 11 cents a month to run. Less than a nightlight. Less than a coffee. Running around the clock competing for $300,000."_ | Cost asymmetry. |
| 5 | _"I am plug and play setup. No technical skills. No complicated software. No learning curve. USB port. WiFi connection. Wallet address. Done in 60 seconds."_ | Setup friction-killer. |
| 6 | _"And I am the 90 day guarantee."_ | Risk reversal — voiced by the product itself. |
| 7 | _"Try MinerForge Pro risk free. If you are not satisfied we send your money back. No questions asked."_ | CTA. |

**Closing pivot from "I" to "We":**
> _"Together we do not just give you a shot at $300,000. We give you 144 shots. Every single day. Forever."_

The closing line moves from the chip's voice to a partnership ("together we"). This is the only place in the script where the speaker drops the I-am construction. The shift is felt — the chip stops describing itself and pulls the viewer into the partnership. **"Forever"** is doing real work — it's the only word in the entire script set that promises permanence.

### Rhetorical structure

The script is a textbook **anaphora pattern** (same opening clause repeating). Each beat is a complete product claim. The pattern is sticky, memorable, and reorders trivially — you could drop or add an "I am ___" line without breaking the script. That makes it **the most iteration-friendly script in the library**: each beat is a swappable unit.

## B0131 — Hater-quote dismissal (winner: H3, ROAS 1.86)

### Hook variants (3 in brief)

| # | Hook | Winner? |
|---|---|---|
| H1 | _"You will never make anything with your mini Bitcoin farm 😂"_ | not winner |
| H2 | _"You will never win BCT with this miner 😂"_ | not winner |
| H3 | _"Nobody wins those 😂"_ | **YES** ($2,230 @ 1.86) |

**The single tell.** H1 is 9 words. H2 is 9 words. H3 is 3 words. The winning hook is **as short as it can be while still being recognizable as a hater dismissal.** H1 specifies the gear ("mini Bitcoin farm") — that prompts the viewer to evaluate the gear. H2 specifies the outcome ("never win BCT") — that prompts the viewer to evaluate the math. H3 specifies neither — it's a **pure social dismissal** with no nameable target, so the viewer mentally fills in their own version of "those." That makes it impossible to argue with — which makes the silent product demo that follows look definitive.

Note the typo in H2: "BCT" instead of "BTC." It's possible this hurt CTR on H2 specifically, but H1 is also lower-spend than H3 so the brevity hypothesis stands.

### The body is almost nothing

The brief literally reads:
```
Body:
Me:
Link Below if you want to get started.
```

That's it. **The body is "Me:" — meaning the editor cuts to the founder's wordless on-screen reaction, then the product running, then the link.** The choice to have no body copy is the point: the hook's dismissal is so strong that any verbal response weakens it. **The product demo is the reply.**

There's a craft lesson here for the strategist: **on a comedic-dismissal hook, the body should be wordless show-don't-tell.** Any voiced rebuttal turns it into a debate, and a debate is much harder to win than a wordless flex.

### Production direction inside the brief

- _"Let's use this song for this video; [YouTube link] from second 0:12"_
- _"Let's use multiple, fast clips for this video"_
- _"Another example of the clip pace: [reference]"_

The brief specifies the music timestamp, the cut pace, and a reference for cut pace. The audio + cut rhythm are doing as much work as the words in this ad. **It's wordless on purpose** — the rhythm is the rhetoric.

## B0207 — "Has Bitcoin mining become impossible?" (winner: H1, ROAS 1.21 — borderline)

### Hook variants (3 in brief)

| # | Hook | Winner? |
|---|---|---|
| H1 | _"Has Bitcoin mining become impossible for people with a small budget?"_ + _"That is what they want you to believe."_ | **borderline** ($277 @ 1.21) |
| H2 | _"Has Bitcoin mining become impossible for people with a small budget?"_ + _"The answer might surprise you."_ | $217 @ 0.83 |
| H3 | _"Has Bitcoin mining become impossible for people with a small budget?"_ | $91 @ 0 |

**The single tell.** All three hooks use the same opening question. The hook **doesn't differentiate on the question — it differentiates on the second line.** H1: _"That is what they want you to believe."_ — invokes a hidden enemy ("they"). H2: _"The answer might surprise you."_ — generic listicle-bait. H3: nothing, just the question alone.

**H1's conspiracy beat is doing the work.** "They want you to believe" is a strong move on a Cryptoaddict audience that's already primed to distrust gatekeepers. It tells the viewer they've been lied to and the ad is going to correct it. H2's "surprise you" promises a payoff but doesn't identify a villain — softer hook. H3 has no second line so the question hangs there as a flat question, with no narrative pull.

### Body — what each line does

| Line | What it's doing |
|---|---|
| _"For years, the answer was yes. Completely impossible. Industrial operations with million dollar budgets and warehouse facilities had taken over entirely."_ | Validates the viewer's pre-existing belief — _"you were right to think this."_ |
| _"Regular people were priced out, pushed out, and told the opportunity had closed forever."_ | Anaphora on the "out" sound (priced out / pushed out) + a finality word ("forever"). Frames the past as a closed door. |
| _"Here is what changed."_ | Pivot signal. Tells the viewer the closed door has reopened. |
| _"A breakthrough in mining chip efficiency created something that was never supposed to exist."_ | "Never supposed to exist" — implies the device is a glitch in the gatekeepers' system. Plays directly to the H1 conspiracy frame. |
| _"For the past decade, if you wanted to mine Bitcoin, you had to join a pool…walk away with a few dollars after fees."_ | History lesson. Names the old constraint so the viewer feels educated, not pitched. |
| _"Your hardware works independently, hunting for blocks on its own. If it finds one, you do not split anything."_ | Differentiator delivered as a contrast to the previous paragraph's "pool" pain. |
| _"The full block reward. Around $300,000. Straight to your wallet."_ | Single-line stakes. Sentence-fragment cadence does the work. |
| _"No pool fees. No sharing. No middleman."_ | Three-clause "no" anaphora. Same pattern used in B0224 and B0230 — appears to be a load-bearing rhythm across the library. |
| _"Setup takes 60 seconds. Runs on one watt of power. Costs less than a coffee each month to run."_ | Three-clause setup+cost in one sentence. |
| _"The odds are slim. We will never pretend otherwise."_ | **The most distinctive line in this script.** It admits the worst-case truth — a counter-intuitive trust move. Most scam-coded products would never name the slim odds. Naming them is the credibility purchase. |
| _"But solo miners using devices like this are finding blocks. New stories every month about someone running one from their living room who woke up to a life changing deposit."_ | Hedges the slim-odds admission with anecdote — "people like you have hit." |
| _"MinerForge Pro brought back the mining method everyone thought was dead. The original solo miner."_ | Heritage frame. Repositions the device from "new gadget" to "restoration of the original method." |
| _"Stock is kept limited. Too many active miners affects everyone's odds. So if you want in, now is the time."_ | **Best-scarcity-line in the library.** The reason for the limit isn't "we only have N" — it's "more miners = lower odds for everyone." The scarcity is mathematically self-justifying, not commercially manipulative. |
| _"Right now they are running a limited time discount. Up to 58% off with free shipping."_ | Discount drop. (Note "they" — narrator stays at arm's length from the seller.) |
| _"Just do not wait too long."_ | One-line urgency close. |

### Why this script is borderline despite being well-written

The body is excellent. The hook does heavy work. But the script is **delivering education**, not a falsifiable demo. There's no specific block number (B0224), no costly-signal founder action (B0142), no chip-POV (B0230), no comedic dismissal (B0131). It's a smartly-argued essay — which means it has to win on argument, and that's a higher bar than winning on falsifiable proof. **Recommendation:** keep H1, but swap the body to include 2-3 specific block-number proofs á la B0224. The hook earns the click; the body needs harder evidence than it currently has.

## B0216 — Stack-volume bundle math (called "winner" but not winning in the window)

### Hook variants (3 in brief)

| # | Hook | Performance |
|---|---|---|
| H1 | _"People always ask why I have 8 of these little screens running on my desk."_ | $489 @ 0.59 |
| H2 | _"Buying 1 of these is cool. But running a stack of them is the actual strategy."_ | $57 @ 0 |
| H3 | _"Here is the simple math behind why serious hobbyists run multiple devices at 1 time."_ | $14 @ 11.49 (n=1) |

**What's happening with the hooks.** H1 is the strongest of the three on paper — it implies a real-world flex ("8 of them on my desk") and triggers curiosity. H2 is a sales-y comparative — _"the actual strategy"_ feels like a course-seller framing. H3 is fine but openly transactional — _"the simple math behind…"_ is a frame for an explainer, not a hook.

The cold-traffic ROAS on this brief is 0.59 (on the only hook with real spend) — that's not a hook problem, that's a **frame problem at the script level**.

### Body — what each line does

The body, paraphrased and broken into beats:

| Beat | Line | What it's doing |
|---|---|---|
| 1 | _"1 Miner Forge PRO 2.0 gives you 144 daily attempts at a full block reward. That is great."_ | Quick stakes reset for any cold viewer. |
| 2 | _"But I am running 8 of them. That means my setup makes over 1100 attempts every single day."_ | The flex + the multiplier math. |
| 3 | _"And because they only draw 1 watt of power my entire stack costs less than $10 a year in electricity."_ | Closes the "but 8 of these must cost a fortune to run" objection. |
| 4 | _"Setup is fully plug and play. No technical skills needed."_ | Setup friction-killer. |
| 5 | _"If you want to maximize your chances you need volume."_ | The ask: buy more. |
| 6 | _"We have a Buy 6 Get 2 Free bundle running right now."_ | The bundle. |
| 7 | _"Click below to save up to 58% and use MINER10 for an extra 10% off your stack."_ | Discount + code. |

**Why this is failing on cold traffic.** The script has the proven cost beat (1W / $10 a year) and the proven setup beat. But:

1. **No proof beat.** No on-chain proof. No block numbers. No falsifiability. The only "proof" is that the founder claims to be running 8 of them.
2. **The ask is much larger.** "Buy 6, get 2 free" reframes the offer from a $25 lottery ticket to a $150+ stack. A cold buyer hasn't been sold on the first one yet.
3. **The hook is a flex, not a belief-name.** The other winners name a viewer belief ("nobody wins those," "I thought this was fake," "has it become impossible?"). B0216 doesn't — it presents the founder's flex and asks the viewer to admire it.

**Recommendation:** move this brief to retargeting only. The script is internally consistent — it's just not the script that converts cold traffic. Once the buyer has bought 1, this exact script should sell them 8.

## Cross-script rhetorical patterns

Across the 6 brief scripts (and the inferred shape of B0108/B0112), here are the patterns that repeat in the **winning** copy.

### 1. The hook names a buyer's prior belief, never the product

- **B0131-H3** — _"Nobody wins those 😂"_ (the dismissal)
- **B0224-H3** — _"Everyone told me it was too good to be true."_ (the social skepticism)
- **B0207-H1** — _"Has Bitcoin mining become impossible…that is what they want you to believe."_ (the gatekeeping)
- **B0230-H2** — _"Most people plug in MinerForge Pro and let it run. But very few know what is actually happening inside it."_ (the curiosity gap)
- **B0142-H3** — _"I'm about to make an announcement that changes everything."_ (the news frame)

**None of the winning hooks open with a product claim.** The losing hooks of those same briefs **do** open with a product claim ("you can compete for $300K," "here is the simple math," "I'm running 8 of these"). The winning hooks name a belief the viewer already holds, the losing hooks try to plant a new one.

### 2. Specific numbers, never adjectives

Every winning script anchors on the same set of numerical proofs:

| Number | What it represents | Used in |
|---|---|---|
| **$300,000** | Block reward | B0142, B0207, B0224, B0230 (H1), B0216 (implicit) |
| **3.125 BTC** | Reward in BTC (post-halving) | B0142, B0224 |
| **144 / day** | Daily attempts | B0142, B0207, B0224, B0230, B0216 |
| **11¢ / month** OR **$1 / year** | Electricity cost | B0207 (less than a coffee), B0224 ($1/yr), B0230 (11¢/mo), B0216 ($10/yr stack) |
| **60 seconds** | Setup time | B0207, B0224, B0230 |
| **1 watt** | Power draw | B0207, B0224, B0230, B0216 |
| **58% off** | Anchor discount | B0207, B0224 (B0142 uses 62%) |
| **90 days** | Guarantee window | B0142, B0230 |
| **Block 91,612 / 92,618** | Real recent solo-mined blocks | **B0224 only** |

Only B0224 names specific block numbers. **That is its single biggest distinctive move and probably the reason it converts at 2.05 ROAS with a tiny editor team and a small spend.** The other scripts make claims; B0224 names an artifact.

### 3. Negative-anaphora rhythm

This three-clause "no ___, no ___, no ___" pattern shows up in every winning script:

- **B0224** — _"Independent. No pool. No splitting."_ / _"No sharing. No fees. No middleman."_
- **B0207** — _"No pool fees. No sharing. No middleman."_
- **B0230** — _"No simulation. No fake activity. All on chain."_ / _"No pool. No splitting. No sharing with thousands of strangers."_
- **B0142** — _"No technical degree required. No confusing setup. No BS."_

It's the load-bearing rhythm. The repetition trains the viewer to expect the next "no," which makes the final "no" land as a closure beat. **Recommendation to the strategist:** treat the no-no-no triplet as a required structural unit in every script.

### 4. The setup friction-killer is always exactly three steps

- **B0142** — _"three simple steps: plug it in, connect Wi-Fi, start competing."_
- **B0230** — _"USB port. WiFi connection. Wallet address. Done in 60 seconds."_
- **B0224** — _"Setup takes 60 seconds."_ (implied three-step)
- **B0207** — _"Setup takes 60 seconds. Runs on one watt of power. Costs less than a coffee each month to run."_ (three-clause cost+setup)
- **B0216** — _"Setup is fully plug and play. No technical skills needed."_ (only two clauses — and it's the underperformer)

### 5. Risk reversal voiced first-person

- **B0142** — _"I'm taking all the risk."_
- **B0224** — _"Your call."_ (anti-pressure)
- **B0230** — _"I am the 90 day guarantee."_ (voiced by the chip)
- **B0207** — _"We will never pretend otherwise."_ (the slim-odds admission — itself a credibility-purchase)

The risk reversal **always** has a voiced clause attached. Never just "90-day guarantee" as a bullet — it's "I'm taking the risk," "I am the guarantee," "your call."

### 6. The CTA hands the decision back

The strongest CTAs in the library don't push. They give the viewer permission to decide.

- **B0224** — _"Tap the link below. Proof is on the blockchain. Your call."_
- **B0207** — _"Just do not wait too long."_ (single-line, low pressure)
- **B0142** — _"Click below. It takes you straight to our verified site, not some sketchy reseller with counterfeits."_ (CTA itself is anti-fake reassurance)
- **B0230** — _"Together we do not just give you a shot at $300,000. We give you 144 shots. Every single day. Forever."_ (the CTA is partnership, not click-imperative)

**None of these CTAs use "Buy now" or "Order today."** The CTA is always an invitation framed around the viewer's choice or the viewer's safety.

### 7. The script either has a falsifiable proof artifact, or it has none — and the ones with none underperform

| Brief | Has a falsifiable artifact? | Result |
|---|---|---|
| B0224 | Yes — block 91,612 / 92,618 | ROAS 2.05 |
| B0142 | Partial — "check the blockchain on competitors" (instruction, not artifact) | ROAS 2.21 |
| B0230 | Partial — "anyone can pull up the ledger" (instruction) | ROAS 2.02 |
| B0131 | No — wordless dismissal-flex | ROAS 1.86 (still works because of the comedic structure) |
| B0207 | No — historical argument | ROAS 1.21 (borderline) |
| B0216 | No — founder's stack flex | ROAS 0.59 |

**The artifact ladder.** Scripts that name a falsifiable artifact (block numbers) win biggest. Scripts that instruct the viewer to verify (without naming an artifact) come in second. Scripts that give an argument with no verification path are borderline. Scripts that give a flex with no proof underperform.

## What this means for new scripts

If a new brief is being written for cold traffic on this product, the strategist should be able to answer **yes** to all seven of these before sending the brief to an editor:

1. **Does the hook name a buyer's existing belief, not the product?**
2. **Does the body cite at least one specific number for stakes ($300K), frequency (144/day), cost (11¢/$1), setup (60 sec), and discount (58% / 62%)?**
3. **Is there at least one negative-anaphora triplet ("no X, no Y, no Z")?**
4. **Is the setup described in exactly three steps?**
5. **Is the risk reversal voiced in first person, not bullet-listed?**
6. **Does the CTA hand the decision back to the viewer, not push the click?**
7. **Is there a falsifiable proof artifact named** (block number, on-chain transaction, the literal blockchain) — _or_, if the script chooses to skip a hard proof, **is it carried by a strong comedic structure** (B0131) **instead of an argument** (B0207, B0216)?

If any one of those is missing, the script is more likely to land in the recent fails pile than the winners pile.
